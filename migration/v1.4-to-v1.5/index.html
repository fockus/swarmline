    Migration guide: swarmline v1.4 → v1.5¶

    This guide covers every user-visible change between swarmline 1.4.x and 1.5.0. Most of v1.5.0 is additive — new runtimes, new tools, multimodal input, parallel agents. The three changes that warrant attention are:

    1. The default runtime flipped from claude_sdk to thin.
    2. Python 3.10 is no longer supported. Minimum is now >=3.11.
    3. serve.create_app(allow_unauthenticated_query=True) requires explicit host= (security audit closure, P2 #5).

    If your code pins runtime= explicitly, your interpreter is already >=3.11, and you do not use the unauthenticated /v1/query endpoint, the upgrade is drop-in.

    TL;DR¶

    • Pin runtime= explicitly in AgentConfig to avoid surprise from the new default.
    • Upgrade your interpreter to Python 3.11 or newer.
    • If you use serve.create_app(allow_unauthenticated_query=True), add host="127.0.0.1" (or another loopback) — the legacy signature without host= is now a hard error.
    • AgentConfig.max_thinking_tokens is deprecated — start migrating to thinking= (typed).
    • All other changes are purely additive: opt in when you need them.

    Breaking changes¶

    1. Default runtime: claude_sdk → thin¶

    Before (v1.4.x):

    from swarmline.agent import Agent, AgentConfig

agent = Agent(AgentConfig(system_prompt="You are a helpful assistant."))
# Implicitly runtime="claude_sdk" — required `claude-agent-sdk` to be installed.

    After (v1.5.0):

    from swarmline.agent import Agent, AgentConfig

agent = Agent(AgentConfig(system_prompt="You are a helpful assistant."))
# Now runtime="thin" — uses ThinRuntime with multi-provider support.

    Why this changed: the in-tree thin runtime now reaches feature parity with the Claude Agent SDK adapter on hooks, tool policy, subagents, slash-commands, native tool calling, multimodal, and session resume. It is a better default because it works with Anthropic, OpenAI, Google, DeepSeek, and OpenAI-compatible providers out of the box, without requiring the claude_agent_sdk package on PyPI.

    How to migrate:

    If you depend on claude_sdk-specific behaviour, pin it explicitly:

    agent = Agent(AgentConfig(
    system_prompt="...",
    runtime="claude_sdk",   # restores v1.4 behaviour
))

    Otherwise, the new thin default works for most users; we recommend pinning it anyway to make the choice durable across future major releases:

    agent = Agent(AgentConfig(
    system_prompt="...",
    runtime="thin",         # explicit lock to current default
))

    2. Python 3.10 is no longer supported¶

    pyproject.toml now declares requires-python = ">=3.11" and the publish workflow no longer tests against 3.10. Upgrade your interpreter before installing swarmline 1.5.0.

    3. serve.create_app requires explicit host= for unauthenticated mode¶

    Audit closure P2 #5 (HIGH severity). When allow_unauthenticated_query=True and auth_token=None, the v1.4.x signature create_app(agent, allow_unauthenticated_query=True) (without host=) emitted a deprecation warning but still constructed the app. This permitted operators to combine the legacy call with uvicorn --host 0.0.0.0 and silently expose unauthenticated /v1/query publicly. v1.5.0 graduates the warning to a hard ValueError.

    Before (v1.4.x — deprecated, still worked):

    from swarmline.serve.app import create_app

app = create_app(agent, allow_unauthenticated_query=True)
# DeprecationWarning emitted; app still constructed.

    After (v1.5.0 — raises ValueError):

    from swarmline.serve.app import create_app

# Local-only mode: explicit loopback host
app = create_app(
    agent,
    allow_unauthenticated_query=True,
    host="127.0.0.1",   # required since v1.5.0
)

# Production mode: token auth (host enforcement bypassed by auth)
app = create_app(agent, auth_token="<server-token>")

    Allowed loopback hosts: 127.0.0.1, localhost, ::1. Any other host raises ValueError at factory time. If you intentionally need an unauthenticated public listener, configure auth_token and an external proxy with its own auth layer instead — there is no supported "unauthenticated + public" combination by design.

    4. LLM error message language¶

    ThinLlmError messages emitted from runtime/thin/errors.py are now in English. This is only noticeable if you grep on the literal string in logs or tests:

    Before After
    Ошибка LLM API (anthropic): … LLM API error (anthropic): …

    If you have log filters or tests that match the Russian string, update them.

    Deprecations (still functional, will be removed in a future release)¶

    AgentConfig.max_thinking_tokens → thinking=¶

    The dedicated integer field is being phased out in favour of a structured thinking parameter that accepts either a dict or one of the typed dataclasses exported from swarmline.runtime.types.

    Old (still works in 1.5.0):

    AgentConfig(system_prompt="...", max_thinking_tokens=4096)

    New (preferred):

    # Dict form — accepted but not type-checked.
AgentConfig(
    system_prompt="...",
    thinking={"type": "enabled", "budget_tokens": 4096},
)

# Typed form — type-checked, IDE-friendly.
from swarmline.runtime.types import ThinkingConfigEnabled

AgentConfig(
    system_prompt="...",
    thinking=ThinkingConfigEnabled(type="enabled", budget_tokens=4096),
)

    Other variants:

    • ThinkingConfigAdaptive(type="adaptive") — let the model decide.
    • ThinkingConfigDisabled(type="disabled") — explicitly disable thinking.

    The thinking= argument also takes precedence when both are passed.

    New features (additive — no migration required)¶

    You can adopt any of these incrementally. None of them changes existing behaviour unless you opt in.

    Foundation filters (Phase 11)¶

    from swarmline import ProjectInstructionFilter
from swarmline.runtime.types import RuntimeConfig

config = RuntimeConfig(
    model="sonnet",
    input_filters=(ProjectInstructionFilter(),),
)

    ProjectInstructionFilter walks up from the current working directory, loading CLAUDE.md, AGENTS.md, RULES.md, and GEMINI.md into the system prompt — the same convention that Claude Code uses.

    Tool surface expansion (Phase 12)¶

    • web_fetch now respects web_allowed_domains / web_blocked_domains.
    • read_mcp_resource builtin tool reads MCP resources by URI.
    • list_resources / read_resource MCP helpers with TTL caching.

    Conversation compaction (Phase 13)¶

    When the message history grows past a threshold, ThinRuntime can collapse it through three levels: tool-result truncation → LLM summarisation → emergency truncation. Configure via CompactionConfig:

    from swarmline.runtime.thin.compaction import CompactionConfig

config = RuntimeConfig(
    model="sonnet",
    compaction=CompactionConfig(
        threshold_tokens=80_000,
        target_tokens=20_000,
    ),
)

    Session resume (Phase 14)¶

    from swarmline.agent import Agent, AgentConfig

agent = Agent(AgentConfig(system_prompt="..."))
session_id = "user-42-thread-1"

# First conversation — auto-persisted to JSONL.
async with agent.conversation(session_id=session_id) as conv:
    await conv.send("Hi, I'm Anton.")

# Later, in a new process, resume the same session.
async with agent.conversation(session_id=session_id) as conv:
    reply = await conv.send("Do you remember my name?")

    JsonlMessageStore writes append-only JSONL files (SHA-256 filenames, corrupted-line resilience). Auto-compaction kicks in when the session grows past the threshold.

    Thinking events (Phase 15)¶

    Models that emit extended-thinking blocks now surface them as RuntimeEvent.thinking_delta events in addition to the regular text deltas. The LlmCallResult envelope captures thinking content end-to-end. AnthropicAdapter extracts the dedicated thinking blocks; other providers fall through transparently.

    Multimodal input (Phase 16)¶

    import base64
from swarmline.domain_types import Message, TextBlock, ImageBlock

with open("diagram.png", "rb") as f:
    image_b64 = base64.b64encode(f.read()).decode()

message = Message(
    role="user",
    content="What's in this image?",
    content_blocks=[
        TextBlock(text="What's in this image?"),
        ImageBlock(data=image_b64, media_type="image/png"),
    ],
)

    Message.content_blocks is an additive field — the existing text field still works and is treated as a single TextBlock internally. Provider converters target Anthropic, OpenAI, and Google formats. PDF and Jupyter notebook extractors are bundled.

    Parallel agents (Phase 17)¶

    from swarmline.runtime.thin.subagent_tool import SubagentToolConfig

agent = Agent(AgentConfig(
    system_prompt="...",
    subagent_config=SubagentToolConfig(
        isolation="worktree",   # each spawned subagent gets its own git worktree
    ),
))

    When isolation="worktree", every spawn_agent invocation runs in a fresh git worktree on a temporary branch and is cleaned up automatically when no changes were made. Spawned agents emit RuntimeEvent.background_complete when they finish; the new monitor_agent builtin tool lets the parent agent track their progress.

    New runtimes¶

    • runtime="openai_agents" — OpenAI Agents SDK (Codex + OpenAI models). See the agent SDK for deep tool integration and Codex-specific features.
    • runtime="pi_sdk" — Pi SDK via a Node.js bridge. Useful when you want Pi's planning model in the loop.

    Both runtimes are registered through the standard RuntimeRegistry. Install their optional extras (swarmline[openai-agents], swarmline[pi-sdk]) to pull the dependencies in.

    Agent packs¶

    Declarative agent bundles let you ship a packaged agent definition (system prompt, tools, runtime config) as a single resource:

    from swarmline.agent_pack import AgentPackResolver

resolver = AgentPackResolver()
pack = resolver.resolve("github://acme/agents/qa-bot@v1")
agent = pack.build_agent()

    See docs/agent-pack.md for the resource layout.

    JSONL telemetry sink¶

    from swarmline.observability.event_bus import InMemoryEventBus
from swarmline.observability.jsonl_sink import JsonlTelemetrySink

bus = InMemoryEventBus()
sink = JsonlTelemetrySink(path="logs/events.jsonl", event_bus=bus)
await sink.start()  # auto-subscribes to bus, writes append-only JSONL

    I/O is wrapped in asyncio.to_thread(); concurrent writes are serialised with an asyncio.Lock. Built-in key-name redaction strips secrets from records before they hit disk.

    Coding profile¶

    from swarmline.runtime.thin.coding_profile import CodingProfileConfig

agent = Agent(AgentConfig(
    system_prompt="You write Python.",
    coding_profile=CodingProfileConfig(),  # enables sandbox + web + thinking with hardened policy
))

    Bug fixes that may affect users¶

    Test isolation (C-1 / C-3)¶

    If you ran swarmline-using tests in your CI and saw weird "logs end up on stdout" or "tests pollute each other's loggers" issues — those were tracked as C-1 and C-3. Fixed in 1.5.0 by:

    • removing logging.basicConfig(force=True) from observability/logger.py (now idempotent and only configures when no handlers exist),
    • routing stdlib logging to stderr (was stdout), so swarmline … --json writes clean JSON to stdout,
    • routing structlog to stderr as well via PrintLoggerFactory(file=sys.stderr),
    • adding a _reset_root_logger autouse fixture in tests/conftest.py for clean test cross-pollination.

    If your application configured logging through swarmline's helper and relied on the old force=True behaviour, you may need to call logging.basicConfig(...) in your own bootstrap.

    Non-blocking telemetry (C-2)¶

    JsonlTelemetrySink.record() no longer blocks the event loop on writes. If you have measured event-loop stalls during heavy logging in 1.4, those should be gone. No code change required.

    Verifying the upgrade¶

    pip install --upgrade swarmline
python -c "import swarmline; print(swarmline.__version__)"  # 1.5.0

# Run your test suite — most users see no diff.
pytest -q

# If something broke around `runtime=`, the most common cause is the default flip.
# Pin it explicitly and re-run.
