# Plan 16-01: Multimodal Input

## Summary

Add ContentBlock types (TextBlock, ImageBlock) to domain, extend Message with additive content_blocks field, implement provider-specific vision conversion in adapters, add image detection in read tool, and optional PDF/Jupyter extractors.

## Tasks

<task id="1" name="ContentBlock Domain Types + Message Extension" owner="dev-domain" critical="no">
<action>
1. Create `ContentBlock` types in `src/swarmline/domain_types.py`:
   - `TextBlock(text: str)` — frozen dataclass
   - `ImageBlock(data: str, media_type: str)` — frozen dataclass (data = base64)
   - `ContentBlock = TextBlock | ImageBlock` — type alias

2. Add `content_blocks: list[ContentBlock] | None = None` to `Message` in `domain_types.py`:
   - Additive field — content: str stays unchanged
   - None default for backward compat (L-004)
   - Update `to_dict()` to serialize content_blocks when present

3. Update `_messages_to_lm` in `runtime/thin/helpers.py`:
   - When `msg.content_blocks` is set, include them in the dict as `"content_blocks"` key
   - When None, behavior unchanged (just `{"role", "content"}`)

4. Update `_filter_chat_messages` in `runtime/thin/llm_providers.py`:
   - Preserve content_blocks from original message dict
   - Pass through to adapters

5. Export ContentBlock types from `src/swarmline/__init__.py`:
   - TextBlock, ImageBlock, ContentBlock (from runtime/types.py re-export)
</action>

- _filter_chat_messages preserves content_blocks
</verify>

<task id="2" name="Provider Adapter Content Conversion" owner="dev-infra" critical="yes" depends="1">
<action>
1. Create `_convert_content_blocks_anthropic(content_blocks)` helper in `llm_providers.py`:
   - TextBlock → `{"type": "text", "text": block.text}`
   - ImageBlock → `{"type": "image", "source": {"type": "base64", "media_type": block.media_type, "data": block.data}}`
   - Returns `list[dict]` suitable for Anthropic messages API

2. Extend `AnthropicAdapter.call()`:
   - Check if messages have content_blocks (via `_content_blocks` key in message dict)
   - When present: build multi-part content array using converter
   - When absent: use content string as before (backward compat)

3. Create `_convert_content_blocks_openai(content_blocks)` helper:
   - TextBlock → `{"type": "text", "text": block.text}`
   - ImageBlock → `{"type": "image_url", "image_url": {"url": f"data:{block.media_type};base64,{block.data}"}}`

4. Extend `OpenAICompatAdapter.call()`:
   - Same pattern: check content_blocks, convert when present

5. Create `_convert_content_blocks_google(content_blocks)` helper:
   - TextBlock → `{"text": block.text}`
   - ImageBlock → `{"inline_data": {"mime_type": block.media_type, "data": block.data}}`

6. Extend `GoogleAdapter.call()`:
   - Same pattern: check content_blocks, convert when present

7. Update `_filter_chat_messages` to handle dict messages with content_blocks:
   - Extract content_blocks from message dict when present
   - Store as serialized list in output dict under `_content_blocks` key
</action>
<verify>
- Anthropic conversion produces correct vision block format
- OpenAI conversion produces correct image_url format
- Google conversion produces correct inline_data format
- All adapters work without content_blocks (backward compat)
- All existing tests pass
</verify>

<task id="3" name="Read Tool Image Detection + PDF/Jupyter Extractors" owner="dev-domain" critical="no" depends="1">
<action>
1. Add `read_file_bytes(path: str) -> bytes` to `SandboxProvider` protocol in `tools/protocols.py`:
   - Default implementation reads binary file
   - LocalSandbox implementation: `safe_path.read_bytes()`

2. Create image detection in `tools/builtin.py` `_create_read_executor`:
   - Check file extension: `.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`
   - For images: read as bytes via `sandbox.read_file_bytes(path)`, base64 encode
   - Return JSON: `{"status": "ok", "type": "image", "media_type": "image/png", "data": "<base64>"}`
   - For text files: behavior unchanged

3. Create `src/swarmline/tools/extractors.py` module:
   - `extract_pdf(path: str) -> str`: lazy import pymupdf4llm, convert to markdown
   - `extract_jupyter(path: str) -> str`: lazy import nbformat, extract cell text
   - Both raise ImportError with clear install instructions if dep missing
   - Both raise FileNotFoundError if path missing

4. Integrate extractors in read tool:
   - `.pdf` extension → try extract_pdf, return text content
   - `.ipynb` extension → try extract_jupyter, return text content
   - ImportError → return error message with install instructions

5. Update `_READ_SCHEMA` to document image/PDF/Jupyter support in description
</action>
<verify>
- Read tool detects PNG/JPG and returns base64 image data
- Read tool detects PDF and extracts markdown text (when pymupdf4llm available)
- Read tool detects Jupyter and extracts cell text (when nbformat available)
- Missing optional deps produce clear error messages
- Text files unchanged
- All existing tests pass
</verify>
