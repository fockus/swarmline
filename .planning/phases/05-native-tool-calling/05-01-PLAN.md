## Tasks

<task id="5.1" name="NativeToolCallAdapter protocol and types" critical="no">
<action>
1. Create `src/swarmline/runtime/thin/native_tools.py`:
   - `NativeToolCall` frozen dataclass: id, name, args
   - `NativeToolCallResult` frozen dataclass: text, tool_calls, stop_reason
   - `NativeToolCallAdapter` Protocol: `call_with_tools(messages, system_prompt, tools, **kwargs) -> NativeToolCallResult`
   - `toolspecs_to_anthropic(tools: list[ToolSpec]) -> list[dict]` converter
   - `toolspecs_to_openai(tools: list[ToolSpec]) -> list[dict]` converter
   - `toolspecs_to_google(tools: list[ToolSpec]) -> list[dict]` converter

2. Add `call_with_tools()` method to AnthropicAdapter:
   - Convert ToolSpec list to Anthropic format via toolspecs_to_anthropic
   - Call `self._client.messages.create(tools=..., ...)`
   - Parse response: extract text blocks + tool_use blocks
   - Return NativeToolCallResult

3. Add `call_with_tools()` method to OpenAICompatAdapter:
   - Convert to OpenAI format: `{"type": "function", "function": {"name", "description", "parameters"}}`
   - Call `self._client.chat.completions.create(tools=..., ...)`
   - Parse response.choices[0].message.tool_calls

4. Add `call_with_tools()` method to GoogleAdapter:
   - Convert to Google format: `genai.types.Tool(function_declarations=[...])`
   - Call `self._client.aio.models.generate_content(tools=..., ...)`
   - Parse response function calls

5. Add `use_native_tools: bool = False` to RuntimeConfig

6. Write tests FIRST (TDD):
   - test_native_tool_call_dataclass_frozen
   - test_native_tool_call_result_dataclass_frozen
   - test_toolspecs_to_anthropic_format
   - test_toolspecs_to_openai_format
   - test_toolspecs_to_google_format
   - test_anthropic_adapter_call_with_tools (mock SDK)
   - test_openai_adapter_call_with_tools (mock SDK)
   - test_google_adapter_call_with_tools (mock SDK)
</action>

</verify>
<done>

<task id="5.2" name="React strategy integration with parallel execution and fallback" critical="no">
<action>
1. In `react_strategy.py`, add native tool calling path:
   - Check if `config.use_native_tools` is True
   - Check if adapter implements NativeToolCallAdapter (isinstance check)
   - If both: call `adapter.call_with_tools(messages, prompt, tools)` 
   - Parse NativeToolCallResult: if tool_calls, execute in parallel via asyncio.gather
   - If text only (stop_reason="end_turn"): finalize as normal

2. Parallel execution:
   ```python
   if len(result.tool_calls) > 1:
       results = await asyncio.gather(*[executor.execute(tc.name, tc.args) for tc in result.tool_calls])
   ```
   - Emit tool_call_started/finished events for each

3. Strangler Fig fallback:
   - Wrap call_with_tools() in try/except
   - On any exception: log warning, fall back to JSON-in-text call()
   - Continue with existing parse_envelope() path

4. Pass adapter reference to react strategy:
   - ThinRuntime stores adapter reference
   - Pass to run_react() as optional parameter

5. Write tests:
   - test_native_tools_enabled_uses_call_with_tools
   - test_native_tools_disabled_uses_json_in_text (default behavior)
   - test_native_tools_parallel_execution (2 tool calls → asyncio.gather)
   - test_native_tools_fallback_on_error (exception → JSON-in-text)
   - test_native_tools_single_tool_call (1 tool → sequential)
   - test_native_tools_text_only_response (no tool calls → finalize)
   - test_native_tools_adapter_not_supporting (plain LlmAdapter → JSON-in-text)
</action>
