<objective>
Add background agent execution to the subagent infrastructure so child agents can run asynchronously with completion notifications and a monitoring tool.

Purpose: Some subagent tasks are long-running and should not block the parent agent. Background mode lets the LLM spawn an agent, continue working, and receive a notification when the background agent completes (or fails/times out). A monitor tool lets the LLM poll status and accumulated output.

Output: RuntimeEvent gains a `background_complete` factory, SubagentSpec gains `run_in_background` field, ThinSubagentOrchestrator supports fire-and-forget spawning with output buffering and done callbacks, spawn_agent exposes `run_in_background` parameter, and a new `monitor_agent` tool is registered.
</objective>

    ```python
    def create_monitor_executor(
        orchestrator: Any,
    ) -> Callable[[dict[str, Any]], Coroutine[Any, Any, str]]:
        """Create an async callable that monitors a background agent."""
        async def _execute(args: dict[str, Any]) -> str:
            agent_id = args.get("agent_id")
            if not agent_id:
                return json.dumps({"status": "error", "error": "Missing required argument: 'agent_id'"})
            try:
                status = await orchestrator.get_status(agent_id)
                output = orchestrator.get_output(agent_id)
                return json.dumps({
                    "status": status.state,
                    "output": output or "",
                    **({"error": status.error} if status.error else {}),
                    **({"result": status.result} if status.result else {}),
                })
            except Exception as exc:
                return json.dumps({"status": "error", "error": str(exc)})
        return _execute
    ```
