<objective>
Add git worktree isolation to the subagent infrastructure so child agents can safely modify files in parallel without conflicts.

Purpose: When multiple subagents run simultaneously, they currently share the parent's working directory, making parallel file operations dangerous. Worktree isolation gives each subagent its own git worktree with automatic lifecycle management.

Output: SubagentSpec gains an `isolation` field, ThinSubagentOrchestrator creates/cleans worktrees around agent runs, spawn_agent tool exposes the `isolation` parameter to the LLM, and stale worktrees are cleaned on first use.
</objective>

class WorkspaceStrategy(str, Enum):
    TEMP_DIR = "temp_dir"
    GIT_WORKTREE = "git_worktree"
    COPY = "copy"

@dataclass(frozen=True)
class WorkspaceSpec:
    strategy: WorkspaceStrategy
    base_path: str
    branch_template: str = "{agent_name}/{task_id}"
    auto_cleanup: bool = True

@dataclass(frozen=True)
class WorkspaceHandle:
