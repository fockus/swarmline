# Plan 03-01: SubagentTool — spec, executor, ThinRuntime integration

## Goal
Register spawn_agent tool in ThinRuntime so LLM can delegate tasks to child agents.

## Tasks

<task id="3.1" critical="yes">
<name>SubagentTool: spec + types + executor + tests (TDD)</name>
<action>
Create `src/swarmline/runtime/thin/subagent_tool.py`:

1. `SubagentToolConfig` frozen dataclass:
   - max_concurrent: int = 4
   - max_depth: int = 3
   - timeout_seconds: float = 300.0

2. `SUBAGENT_TOOL_SPEC` — ToolSpec with:
   - name: "spawn_agent"
   - description: "Spawn a child agent to execute a subtask"
   - parameters: {"task": str (required), "system_prompt": str (optional, default "You are a helpful assistant"), "tools": list[str] (optional, default all parent tools)}

3. `create_subagent_executor(orchestrator, config, parent_tools, current_depth=0)` → async callable:
   - Check max_depth: if current_depth >= config.max_depth → return JSON error
   - Parse tool args: task (required), system_prompt (optional), tools (optional subset)
   - Filter parent_tools by requested tool names (or use all if not specified)
   - Build SubagentSpec(name=f"subagent-{uuid}", system_prompt=..., tools=filtered_tools)
   - spawn(spec, task) → agent_id
   - asyncio.wait_for(orchestrator.wait(agent_id), timeout=config.timeout_seconds)
   - On timeout → cancel agent → return JSON error
   - On success → return JSON {"agent_id": ..., "status": "completed", "result": output}
   - On exception → return JSON {"agent_id": ..., "status": "failed", "error": str(e)}
   - NEVER raise — all errors → JSON

Create `tests/unit/test_subagent_tool.py` (TDD RED first, then GREEN):
1. test_subagent_tool_spec_has_required_task_param
2. test_subagent_config_defaults — max_depth=3, max_concurrent=4, timeout=300
3. test_subagent_spawns_and_returns_result — mock orchestrator, verify spawn+wait called
4. test_subagent_timeout_returns_error — wait_for raises TimeoutError → JSON error
5. test_subagent_max_depth_exceeded_returns_error — depth >= max_depth → immediate JSON error
6. test_subagent_max_concurrent_exceeded_returns_error — orchestrator raises ValueError → JSON error
7. test_subagent_inherits_parent_tools — requested tools filtered from parent
8. test_subagent_failure_returns_error_json — exception in orchestrator → JSON error, no crash
9. test_subagent_default_system_prompt — no system_prompt → uses default
10. test_subagent_task_required — missing task → JSON error
</action>
